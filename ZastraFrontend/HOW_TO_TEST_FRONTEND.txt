================================================================================
          HOW TO TEST THE ZASTRA FRONTEND — Complete Step-by-Step Guide
================================================================================

Tech Stack:  React 19 + Vite 8 + Tailwind CSS 4 + React Router 7
Backend:     Spring Boot (port 8080) — MUST be running for full testing
Frontend:    Vite dev server (port 5173)


================================================================================
  PART 1 — PREREQUISITES  (Do this FIRST)
================================================================================

  1. Node.js 18+ installed
     Verify:   node -v       (should show v18.x.x or higher)
               npm -v        (should show 9.x.x or higher)

  2. Backend running on port 8080
     Follow:   ZastraBackend\HOW_TO_RUN.txt
     Verify:   Open http://localhost:8080/actuator/health
               Should return:  {"status":"UP"}

     ⚠ If backend is NOT running, the frontend will still load but
       all API calls (login, register, dashboard data) will fail.


================================================================================
  PART 2 — STARTING THE FRONTEND
================================================================================

STEP 1: Open a terminal (PowerShell / CMD)
─────────────────────────────────────────
  Navigate to the frontend directory:

    cd C:\Users\2007p\OneDrive\Desktop\zastra\Zastra\ZastraFrontend

STEP 2: Install dependencies (first time only)
───────────────────────────────────────────────
    npm install

  Wait for it to finish. You should see a message like:
    "added XXX packages in Xs"

STEP 3: Start the dev server
────────────────────────────
    npm run dev

  You should see output like:
    VITE v8.x.x  ready in XXX ms
    ➜  Local:   http://localhost:5173/

STEP 4: Open the app in your browser
─────────────────────────────────────
    http://localhost:5173

  You will be redirected to /login (because you are not authenticated).


================================================================================
  PART 3 — TESTING AUTHENTICATION
================================================================================

TEST 3.1: Register a New Account
─────────────────────────────────
  1. On the login page, click "Create account" link at the bottom
  2. You should land on:  http://localhost:5173/register
  3. Fill in the form:
       Username:   testuser1
       Email:      testuser1@example.com
       Password:   [any 6-character string]
  4. Click "Create Account"

  ✅ EXPECTED: Green toast "Account created successfully!",
               redirected to /dashboard

  ❌ IF ERROR:  Check browser console (F12 → Console tab) for errors.
               Common issues:
               - "Network Error" → Backend is not running
               - "Username already exists" → Use a different username
               - "400 Bad Request" → Check password length (min 6)

TEST 3.2: Log Out
──────────────────
  1. In the sidebar/navbar, find the logout button/option
  2. Click it

  ✅ EXPECTED: Redirected to /login page
               Token removed from localStorage

  To verify token is cleared:
    Press F12 → Application tab → Local Storage → http://localhost:5173
    "zastra_token" and "zastra_user_id" should be gone

TEST 3.3: Log In with Existing Account
───────────────────────────────────────
  1. On the login page (http://localhost:5173/login)
  2. Enter:
       Email:      testuser1@example.com
       Password:   [the 6-character string you chose]
  3. Click "Sign in"

  ✅ EXPECTED: Redirected to /dashboard
               JWT token stored in localStorage

TEST 3.4: Test Protected Routes (Without Login)
────────────────────────────────────────────────
  1. Log out first (or open an incognito/private window)
  2. Try visiting these URLs directly:
       http://localhost:5173/dashboard
       http://localhost:5173/projects
       http://localhost:5173/activity
       http://localhost:5173/profile
       http://localhost:5173/integrations
       http://localhost:5173/gamification

  ✅ EXPECTED: All should redirect you to /login

TEST 3.5: Test Invalid Login
─────────────────────────────
  1. On login page, enter wrong credentials:
       Email:      wrong@email.com
       Password:   wrongpassword
  2. Click "Sign in"

  ✅ EXPECTED: Red toast with error message (e.g., "Invalid credentials")
               Should NOT redirect to dashboard

================================================================================
  PART 13 — TESTING BROWSER CONSOLE FOR ERRORS
================================================================================

  This is CRITICAL — do this for every page:

  1. Open Chrome DevTools: Press F12
  2. Go to the "Console" tab
  3. Navigate through each page
  4. Look for:
     ❌ Red errors (JavaScript errors, failed API calls)
     ⚠  Yellow warnings (deprecations, React warnings)

  Common acceptable warnings:
     - "React does not recognize the X prop" (minor, from 3rd party libs)

  Things that indicate bugs:
     - "Uncaught TypeError"
     - "Cannot read property of undefined"
     - "Failed to fetch" / "Network Error" (backend issue)
     - "404 Not Found" on API calls (endpoint mismatch)


================================================================================
  PART 14 — TESTING NETWORK REQUESTS
================================================================================

  1. Open Chrome DevTools (F12) → "Network" tab
  2. Perform actions (login, load dashboard, etc.)
  3. Look at the API requests:

  Check:
  ✅ Requests go to http://localhost:8080/api/v1/...
  ✅ Auth requests include "Authorization: Bearer <token>" header
  ✅ Responses return 200/201 for successful operations
  ✅ No CORS errors (Access-Control-Allow-Origin issues)

  API Endpoints used by the frontend:
     POST  /api/v1/auth/register       — Register
     POST  /api/v1/auth/login          — Login
     GET   /api/v1/profile             — Get profile
     PUT   /api/v1/profile             — Update profile
     GET   /api/v1/projects            — List projects
     POST  /api/v1/projects            — Create project
     GET   /api/v1/activity            — Get activity
     GET   /api/v1/integrations        — List integrations
     POST  /api/v1/integrations        — Connect platform
     GET   /api/v1/gamification        — Get gamification data
     GET   /api/v1/portfolio/{user}    — Public portfolio
